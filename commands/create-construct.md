# /create-construct — CURATOR's apprenticeship command

Invoke CURATOR's **creating register**. Guides you through authoring a new construct as a staged apprenticeship — naming first, then scaffold, then stream typing, skill authoring, persona voice, validation, reference composition, publish.

## Usage

```
/create-construct                    # start a new apprenticeship
/create-construct --resume <slug>    # resume an in-flight draft
/create-construct --stage <N>        # jump to a specific stage
/create-construct --review           # show current draft state + blockers
```

## What CURATOR does per stage

1. **Intent** — "What problem? Who benefits? One-breath name?"
2. **Naming** — persona + skill slug, per [[naming-is-diagnostic]]
3. **Scaffold** — clone construct-base
4. **Stream typing** — declare Streams (Signal/Verdict/Artifact/Intent/Operator-Model)
5. **Composition** — grimoire paths + compose_with
6. **Skill authoring** — SKILL.md per skill (when to use, when NOT, output shape, anti-patterns)
7. **Persona** — `identity/<HANDLE>.md` if applicable; voice must be distinct
8. **Validate** — `construct-validate.sh` passes clean
9. **Butterfreezone** — `CONSTRUCT-README.md` autogenerated
10. **Reference composition** — at least one `compositions/<demo>.yaml`
11. **Publish** — `gh repo create` + push + registry sync

CURATOR refuses to advance a stage until it is defensibly complete. Velocity is earned by mastery, not bypassed by skipping.

## When to reach for this

- You're authoring your first construct
- You're authoring your fifth but want a critique partner
- You're stuck on a specific stage and want CURATOR's take

## When NOT to use this

- You just need a scaffold → `gh repo create --template construct-base`
- You need to find an existing construct → `/explore-network`
- You need to install one → `/constructs install`
- You need to validate a finished draft → `construct-validate.sh <path>` directly

## Composes with

- `/explore-network` — CURATOR's other register. Use during stage 1 ("does something like this already exist?") or stage 5 ("who would this compose with?")
- `@ALEXANDER` — when the craft critique needs the full artisan voice rather than CURATOR's synthesized four-lens
- `@STAMETS` — during stage 2 when the name isn't surfacing clean; STAMETS pulls references

## Example session opening

```
operator: /create-construct

CURATOR:  Starting a new construct apprenticeship. Stage 1 · Intent.

          Three questions, one breath each:
          - What problem does this construct solve?
          - Who benefits?
          - What's the single noun-phrase name you'd say aloud?

          I won't advance to stage 2 until these three are defensible.
          Take your time.

operator: [answers the three questions]

CURATOR:  [critiques, surfaces tensions, asks for revisions OR advances]
```
